## 四、核心功能实现

### 4.1 代码变更分析 (utils/analyzer.js)
- 使用`git diff`和`git status`获取改动文件列表
- 分析变更类型：新增/修改/删除
- 智能识别文件类型并建议提交类型：
  - 源代码文件 → feat/fix/refactor
  - 测试文件 → test
  - 文档文件 → docs
  - 配置文件 → chore
- 生成变更摘要统计

### 4.2 代码检查和测试 (utils/checker.js)
- **Lint检查**: 执行`yarn lint`，解析输出结果
- **测试执行**: 执行`yarn test`，收集测试结果
- **Husky集成**: 可选择跳过或手动触发pre-commit钩子
- **用户交互**: 检查失败时询问用户是否继续执行
- **结果展示**: 清晰展示检查结果和建议

### 4.3 提交信息生成 (在index.js中实现)
- 基于代码变更分析生成建议的提交信息
- 遵循Conventional Commits规范
- 格式: `<type>: <subject>`
- 提供中文和英文两个版本
- 用户可以查看、编辑或完全自定义
- 使用commitlint验证最终格式

### 4.4 双仓库推送 (utils/pusher.js)
- **目标仓库**:
  - origin: https://github.com/dragonflyoss/console.git
  - alipay: git@code.alipay.com:yjl516914/Dragonfly-console.git
- **推送策略**: 智能选择（根据网络情况和推送结果）
  - 默认串行推送（origin → alipay）
  - 失败时暂停，提示用户选择重试或跳过
  - 支持部分成功场景（一个成功、一个失败）
- **分支处理**: 推送当前分支，自动创建远程分支
- **状态验证**: 推送前验证仓库连接和认证状态

## 五、执行流程

```
1. 前置检查
   ├─ 验证Git仓库状态
   ├─ 检查Node.js和yarn版本
   └─ 验证远程仓库配置

2. 代码变更分析
   ├─ 获取改动文件列表
   ├─ 分析变更类型和影响范围
   └─ 生成变更摘要

3. 代码检查和测试
   ├─ 执行yarn lint
   ├─ 执行yarn test
   └─ 失败时询问用户是否继续

4. 提交信息生成
   ├─ 基于变更分析生成建议信息
   ├─ 展示给用户确认/修改
   └─ commitlint格式验证

5. Git提交操作
   ├─ 添加改动文件到暂存区
   ├─ 执行git commit
   └─ 验证提交结果

6. 双仓库推送
   ├─ 推送到origin (GitHub)
   ├─ 推送到alipay (内部仓库)
   ├─ 智能处理推送失败
   └─ 汇总推送结果
```

## 六、关键技术点

### 6.1 用户交互设计
- 使用彩色输出（chalk）提升可读性
- 使用进度动画（ora）展示执行进度
- 交互式问答（inquirer）收集用户输入
- 清晰的步骤指引和结果反馈

### 6.2 错误处理机制
- 前置检查失败：立即停止并给出具体提示
- 检查失败：询问用户是否继续（强制模式）
- Git操作失败：回滚已执行的操作
- 推送失败：记录失败原因，提供重试选项
- 详细的错误日志和修复建议

### 6.3 配置灵活性
- 支持项目级配置文件（可选）
- 支持命令行参数覆盖默认配置
- 提供合理的默认行为
- 可配置检查项和推送策略
